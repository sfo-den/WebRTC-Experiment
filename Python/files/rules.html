Explains possibility of failure!
This app is heavily using PubNub... which is super easy to use but there is no guarantee of reliability or availability of their service. Personally, I love their service! But hate their limitations!
Pubnub only allow a few hundred free requests. So, there are 70% chances to fail to transfer SDP/ICE/etc. If you fail, try to connect again tomorrow; otherwise download code of this app and use public publish/subscribe keys: 

var socket_config = {
    publish_key: 'demo',
    subscribe_key: 'demo'
};
It means that whenever app fails to connect to other peers; it is because of PubNub socket not opened or their service failed.
This app is heavily using its own two JavaScript wrapper libraries for the sake of simplicity and reusability. Those libraries are: 1) RTCPeerConnection.js and 2) RTCPeerConnection-Helpers.js
RTCPeerConnection-Helpers.js contains code like handling opus codec etc.
RTCPeerConnection.js is a JavaScript wrapper for RTCWeb APIs.
It means that if you download the code of this app and don't reference these two libraries. You'll get no result.
