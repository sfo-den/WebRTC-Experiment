RTCMultiConnection API Reference / "open" method

  1. "open" method allows you start new signaling channel that will act as conference organizer: 
    connection.open('channel-id');
  2. "open" method takes single argument; named as "channel-id": 
    connection.open('channel-id');
  3. If you skip "channel-id"; RTCMultiConnection will use "full-URL" as channel-id: 
    var channel_id = location.href.replace(/\/|:|#|%|\.|\[|\]/g, '');
connection.open(channel_id);

  4. Comparing "open" and "connect" methods:

    var initiator = new RTCMultiConnection();
initiator.open('new-session-007');
		
var participant1 = new RTCMultiConnection();
participant1.connect('new-session-007');

var participant2 = new RTCMultiConnection();
participant2.connect('new-session-007');

var participant3 = new RTCMultiConnection();
participant3.connect('new-session-007');

var participant4 = new RTCMultiConnection();
participant4.connect('new-session-007');

    As you can see; there is only one initiator; however there are 4 session participant/connectors.

    1. "open" method is used merely once for the session-initiator to setup a new channel/session on the signaling server.
    2. "connect" method is called by multiple users to join an existing session/channel.
    3. "connect" doesn't means that you're part of a conference.
