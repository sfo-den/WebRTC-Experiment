RTCMultiConnection API Reference / "connect" method

  1. When you call "connect" method; you're actually setting up signaling channel. You're not part of conferencing room yet. 
    // you can suggest which session/room to join
if(your_are_participant) {
    connection.connect('session-id');
}

// or
if(your_are_participant) {
    connection.connect();
}

A few points to remember

  1. connect method should be called ONLY for initiator i.e. room-owner! 
    if(your_are_participant) {
    connection.connect('session-id');
}

// if you're room owner; then call "open" method instead!
if(your_are_initiator) {
    connection.open('session-id');
}
  2. connect causes delay in initial connection because it waits for onNewSession to be fired. That's why join method is preferred to setup fast connections! 
    // for fast handshake!
if(you_want_to_setup_fast_connection) {
    connection.join('session-id');
}

// for slow connections
if(you_want_to_setup_slow_connection) {
    connection.connect('session-id');
}
  3. onNewSession will be fired only if you're participant and you called connect method. 
    connection.connect('session-id');
connection.onNewSession = function(session) {
    session.join();
};

// if you're room-initiator; then you shouldn't call "connect" method
// that's why "onNewSession" will NEVER be fired for initiator!
