RTCMultiConnection API Reference / "addStream" method

  1. "addStream" method appends media stream in all peer connections: 
  2. // runtime sharing of audio/video among all users
connection.addStream({
    audio: true,
    video: true
});
  3. You can renegotiate data-connection in your existing audio/video/screen session! 
    connection.addStream({
    data: true
});
  4. You can use "peers" object to append between two unique users: 
  5. // runtime sharing of screen among two unique users
// one is you; and other is person whose id is given below
connection.peers['user-id'].addStream({
    screen: true,
    oneway: true
});
  6. Renegotiation is a process allows you modify pre-created peer connections when you want to:
    1. append additional streams
    2. remove existing streams
    3. modify SDP for peers direction or something else
  7. Renegotiation means re-exchanging offer/answer SDP among peers.
  8. Renegotiation means you want to use same peer-connections to append dynamic streams at runtime. 
  9. // runtime sharing of audio/video among all users
connection.addStream({
    audio: true,
    video: true
});

// runtime sharing of screen among two unique users
// one is you; and other is person whose id is given below
connection.peers['user-id'].addStream({
    screen: true,
    oneway: true
});
  10. Remember, renegotiation means: use existing peer connections to negotiate session descriptions.
